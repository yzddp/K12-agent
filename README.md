# 修远 — K12 AI 学习辅导员

陪伴孩子完成 K12 全部 12 年学习的 AI 辅导员。基于 OpenClaw Gateway 运行，飞书 bot 交互。

## 它能做什么

### 对孩子：一个耐心永不枯竭的辅导老师

| 功能 | 描述 |
|------|------|
| **作业辅导** | 发一道题，它不直接给答案，而是用苏格拉底式提问一步步引导孩子自己想出来 |
| **错题拍照分析** | 拍一张批改过的错题照片，自动提取错误类型、沉淀薄弱知识点，累计 3 次同类型错误会自动标记为"需重点关注" |
| **作文批改** | 语文/英语作文结构化批改：审题→结构→语言→立意，不替写整段，只给修改方向 |
| **学习计划** | 根据错题分布和成绩变化，自动生成每日/每周复习任务，动态调整 |
| **情绪感知** | 察觉孩子挫败或焦躁时，换方式引导，记录情绪变化趋势 |
| **积分零花钱** | 完成学习任务、订正错题、进步、获奖自动加积分，攒够可兑换零花钱（家长管控） |

### 对家长：一个不打扰但随时可查的学情管家

| 功能 | 描述 |
|------|------|
| **初始化配置** | 私聊 bot 录入孩子姓名、年级、教材版本、课表，一次设置长期使用 |
| **成绩追踪** | 录入考试成绩，自动分析进步/退步趋势，对比上次同科目 |
| **家长周报** | 每周一推送：新增错题统计、薄弱知识点 Top 5、进步点、下周建议（需配置 cron） |
| **竞赛获奖管理** | 记录孩子获奖情况，自动关联积分 |
| **学期/假期自动切换** | 录入暑假寒假起止日期，假期自动弱化任务，结束前生成"收心计划" |
| **积分兑换审批** | 孩子申请兑换零花钱 → 通知家长确认 → 确认后自动扣减积分 |

### 覆盖学段

- 小学（1-6 年级）：具象化引导、游戏化语气、一步一问
- 初中：逻辑推理训练、概念层讲解
- 高中：综合建模、应试策略、新高考选科参考（官方数据渠道指引）

支持科目：数学、语文、英语、物理、化学、生物、历史、地理、政治

### 身份方案

| 渠道 | 身份 | 可用功能 |
|------|------|----------|
| 私聊 bot | **家长** | 全部功能（配置/报告/积分管理/兑换审批） |
| 群聊 @bot | **孩子** | 辅导答疑/查积分/查错题/复习 |

### 模型选择

**强烈推荐 MiniMax M3**（本 workspace 默认配置）。它支持图片理解、多轮对话推理，经实测在辅导场景下表现稳定。

如果使用其他模型，请确保它支持 **多模态（图片输入）**，并将 `openclaw.json` 中 `model.primary` 改为你的模型 ID。

### 飞书 bot 准备

1. 在 [飞书开发者后台](https://open.feishu.cn/app) 创建企业自建应用
2. 启用 **机器人** 能力
3. 获取 App ID 和 App Secret
4. 开启事件订阅：`im.message.receive_v1`
5. 发布版本并添加至企业通讯录

## 关于错题分析：为什么不直接告诉家长"这道题错在哪"

修远收到错题照片时，内部流程是：

1. **多模态模型描述图片**：从批注中识别错误类型（计算错误/审题偏差/概念混淆等）
2. **沉淀为知识点标签**：将本次错误抽象为"审题弱"、"去括号法则不熟"等标签，写入数据库
3. **持续累加形成画像**：同知识点累计 3 次 → 标记薄弱，纳入学习计划和周报

**为什么要这样做**？

多模态大模型对图片的理解存在固有偏差——它可能看错一个数字、误解一道题的批注、甚至在模型升级前后对同一张图片给出不同的标签。如果直接告诉家长"这道题第 3 步错了"，这个结论可能是错的。

但**错误方向的分类是渐进可靠的**：孩子审题经常出错、分数运算反复丢分——这些模式在 10 次、20 次的持续输入中会稳定浮现，远比单次判断可靠。

所以修远对家长呈现的是**趋势和画像**，而非单次错题的具体诊断。

## 使用场景速查

| "看周报" | 生成学情报告（仅私聊） |
| "我获奖了" | 录入竞赛获奖，自动加积分 |
| "帮我改作文" | 结构化批改语文/英语作文 |
